# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Разработка цифровой платформы мониторинга олимпиадного движения и академических достижений школьников

### 1. ОБЩИЕ ПОЛОЖЕНИЯ
Платформа предназначена для автоматизации сбора, обработки, хранения и аналитики результатов предметных олимпиад школьников. Система обеспечивает сквозное взаимодействие между образовательными организациями (ОО), муниципальными координаторами, специализированными центрами и региональным оператором.

### 2. АРХИТЕКТУРА И МОДЕЛЬ ДАННЫХ
В основу архитектуры базы данных закладывается четырехкомпонентная территориальная иерархия, временное шкалирование по учебным годам и разделение сущностей олимпиады и непосредственного участия ученика.

#### 2.1. Модель «Учебный год»
* **ID** (Уникальный ключ, INT).
* **Название года** (String, например: «2025/2026»).
* **Статус** (Enum: Текущий / Архивный). В системе может быть только один «Текущий» учебный год.

#### 2.2. Модель «АТЕ» (Административно-территориальная единица)
* **ID** (Уникальный ключ, INT).
* **Код АТЕ** (String/INT, уникальный внутренний код управленческого округа).
* **Название АТЕ** (String).
* **Тип АТЕ** (Enum: Обособленный район / Объединенный округ).

#### 2.3. Модель «Район / МСУ»
* **ID** (Уникальный ключ, INT).
* **Код МСУ** (String/INT, официальный код из системы ЕГЭ для синхронизации баз данных).
* **Название МСУ** (String).
* **АТЕ_ID** (Внешний ключ, жесткая связь с моделью АТЕ).

#### 2.4. Модель «Школа» (Образовательная организация — ОО)
* **ID** (Уникальный ключ, INT).
* **Код ОО** (String/INT, уникальный код школы в системе образования / ЕГЭ).
* **Краткое наименование ОО** (String).
* **Полное наименование ОО** (String).
* **Тип ОО** (Enum: Общеобразовательная организация, Лицей, Гимназия и т.д.).
* **Вид ОО** (Enum: С углубленным изучением отдельных предметов и т.д.).
* **Territorial Sign** (Enum: Городская школа / Сельская школа).
* **МСУ_ID** (Внешний ключ, связь с моделью Район/МСУ).
* **Код МСУ** (String/INT, дублирующий сквозной код для оптимизации запросов).
* **АТЕ_ID** (Внешний ключ, связь с моделью АТЕ).
* **Код АТЕ** (String/INT, дублирующий сквозной код для быстрой фильтрации).

#### 2.5. Справочник распределения МСУ по АТЕ для г. Казани:
* МСУ Советский район ➔ АТЕ Советский район (Тип: Обособленный)
* МСУ Кировский район + МСУ Московский район ➔ АТЕ Кировский-Московский (Тип: Объединенный)
* МСУ Авиастроительный район + МСУ Ново-Савиновский район ➔ АТЕ Авиастроительный-Ново-Савиновский (Тип: Объединенный)
* МСУ Приволжский район + МСУ Вахитовский район ➔ АТЕ Приволжский-Вахитовский (Тип: Объединенный)
* Для остальных муниципальных районов РТ применяется схема 1 МСУ = 1 АТЕ.

#### 2.6. Модель «Олимпиада»
* **ID** (Уникальный ключ, INT).
* **Учебный_Год_ID** (Внешний ключ).
* **Предмет** (Enum: Математика, Физика, Русский язык и др.).
* **Этап** (Enum: Школьный, Муниципальный, Региональный).
* **Дата проведения** (Date).
* **Статус олимпиады** (Enum: Планируется / Проверка работ / Апелляция / Результаты опубликованы).

#### 2.7. Модели «Ученик» и «Человеко-олимпиада»
* **Сущность «Ученик»:** Личные данные физического лица.
  * `ID` (Уникальный ключ, INT).
  * `ФИО` (String).
  * `Дата рождения` (Date).
  * `СНИЛС` (String, NULLable).
  * `Школа_ID` (Внешний ключ).
  * `Реальный класс обучения` (INT, 1-11).
  * `Статус ученика` (Enum: Активен / Выпустился / В процессе перевода).
* **Сущность «Человеко-олимпиада» (Заявка на участие):** Логическая единица участия.
  * `Человеко_Олимпиада_ID` (Уникальный ключ, INT).
  * `Ученик_ID` (Внешний ключ).
  * `Олимпиада_ID` (Внешний ключ).
  * `Класс участия` (INT).
  * `Штрихкод` (String, NULLable).
  * `Балл` (Float/NULLable).
  * `Статус результата` (Enum: Участник / Подана апелляция / Призер / Победитель / Дисквалифицирован).
  * `Скан-копия работы` (File/URL, ссылка на загруженный pdf/image файл).

#### 2.8. Модель «Пользователь» (Учетная запись системы)
* **ID** (Уникальный ключ, INT).
* **Логин / Хеш пароля** (String).
* **Роль пользователя** (Enum: Администратор, Супер-координатор Казани, Муниципальный координатор АТЕ, Школьный оператор).
* **АТЕ_ID** (Внешний ключ, NULLable, для Муниципальных координаторов).
* **Школа_ID** (Внешний ключ, NULLable, для Школьных операторов).

#### 2.9. Модуль первоначальной миграции
Реализация механизмов пакетного импорта таблиц Excel/CSV через интерфейс Администратора для первичного наполнения справочников АТЕ, МСУ, ОО (Школ) и пула координаторов.

### 3. РОЛЕВАЯ МОДЕЛЬ И УПРАВЛЕНИЕ ПОЛЬЗОВАТЕЛЯМИ
Самостоятельная регистрация пользователей полностью запрещена. Учетные записи создаются исключительно в рамках строгой нисходящей иерархии: Администратор системы ➔ Муниципальный координатор АТЕ ➔ Школьный оператор.

### 4. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

#### 4.1. Модуль закрытия сезона и перевода учащихся (Ротация)
По окончании олимпиадного цикла запускается пакетный скрипт:
1. Создание нового учебного года со статусом «Текущий» (предыдущий уходит в архив).
2. Выпускники (11 класс) переводятся в статус «Выпустился».
3. Активные ученики (1-10 класс) получают инкремент класса обучения на +1.

#### 4.2. Модуль школьного этапа: Распределение по предметам и ввод данных
Школьный оператор скачивает Excel-шаблон учащихся ОО, проставляет баллы напротив участников по конкретному предмету и загружает обратно. Система создает сущности «Человеко-олимпиада» только для тех строк, где заполнено поле балла.

#### 4.3. Модуль пакетного импорта муниципального этапа (Слепая проверка)
Реализует слияние (JOIN) двух файлов от Городского центра: Файла-ключа (`Человеко_Олимпиада_ID ➔ Штрихкод`) и Протокола жюри (`Штрихкод ➔ Балл`). В интерфейсе выводится интерактивное окно с цветовой индикацией коллизий (Зеленый/Красный/Желтый). Строки «желтой зоны» (участники вне списка) привязываются оператором вручную через веб-интерфейс.

#### 4.4. Модуль аналитики и расчета рейтингов
Расчет локальных рейтингов ОО внутри АТЕ, сквозного рейтинга городского округа Казани по денормализованным кодам и общего республиканского среза в разрезе учебных лет.

#### 4.5. Модуль генерации министерской отчетности
Экспорт ранжированных итоговых протоколов и статистических отчетов по территориям и типам/видам ОО в форматы Excel (.xlsx) и PDF. Обязательна поддержка раздельных срезов по городским и сельским школам.

#### 4.6. Модуль «Онлайн-показ работ» и защита от перегрузок (Anti-DDoS)
* **Вариант 1. Автоматический гостевой доступ (Основной):** Доступ к скан-копии открывается на 48 часов с момента публикации результатов и строго в дневное/вечернее время (с 07:30 до 19:30). Для авторизации участник вводит исключительно свои **ФИО** и **Дату рождения**. Количество попыток ввода с одного IP ограничено (не более 3 попыток, далее — блокировка на 10 минут).
* **Вариант 2. Выгрузка работы школьным оператором (Резервный и пакетный):** Школьный оператор имеет возможность поштучного скачивания скана, а также пакетной выгрузки. При активации пакетного скачивания система упаковывает сканы выбранных учеников или всего класса по предмету в единый ZIP-архив. Маска имени файлов внутри архива: `[Штрихкод]_[ФИО]_[Класс_участия].pdf`.

#### 4.7. Модуль ручного перевода ученика между ОО
Обеспечивает смену привязки ученика к новой школе. Все исторические участия («Человеко-олимпиады»), созданные до даты перевода, навсегда остаются закрепленными за старой ОО для сохранения корректности рейтингов.

#### 4.8. Модуль подачи и обработки апелляций (Муниципальный этап)
Участник в окне онлайн-показа может подать электронное заявление на апелляцию. Муниципальный координатор обрабатывает заявку в своем журнале, выносит решение, и при изменении оценки баллы автоматически пересчитываются во всей системе.

#### 4.9. Модуль автоматизированного регламента очистки и архивации БД
Для предотвращения деградации производительности СУБД внедряется 3-летний жизненный цикл данных:
1. Через 3 года с момента завершения учебного сезона файлы сканов работ (`.pdf`, `.jpg`) безвозвратно удаляются с серверов.
2. Логи авторизации и таблицы блокировок Anti-DDoS очищаются через 30 дней после закрытия показа работ.
3. Персональные данные участников (ФИО, Дата рождения, СНИЛС) в архивных записях старше 3 лет уничтожаются (ФЗ-152). Перед удалением система генерирует и сохраняет агрегированную статичную таблицу исторических итогов по школам и предметам для аналитики.

### 5. ТРЕБОВАНИЯ К ИНТЕРФЕЙСУ И ЛИЧНЫМ КАБИНЕТАМ
Определен состав интерфейсных окон для Администратора (панель ротации, очистки БД), Координатора АТЕ (инфографика, рейтинги), Городского центра Казани (Wizard импорта муниципального этапа) и Школьного оператора (ввод баллов, инструмент пакетной выгрузки сканов в ZIP-архив).
